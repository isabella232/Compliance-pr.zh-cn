## <a name="how-does-microsoft-365-ensure-isolation-between-customer-tenants"></a><span data-ttu-id="e6c9f-114">如何Microsoft 365客户租户之间的隔离？</span><span class="sxs-lookup"><span data-stu-id="e6c9f-114">How does Microsoft 365 ensure isolation between customer tenants?</span></span>

<span data-ttu-id="e6c9f-115">Microsoft 的云服务基于以下假设：所有租户都有可能危害所有其他租户。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-115">Microsoft's cloud services are built on the assumption that all tenants are potentially hostile to all other tenants.</span></span> <span data-ttu-id="e6c9f-116">为了正确地将租户彼此隔离，Microsoft 实施了各种隔离技术和控件。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-116">To properly isolate tenants from one another, Microsoft implements a variety of isolation technologies and controls.</span></span> <span data-ttu-id="e6c9f-117">这些控件旨在防止信息泄露或跨租户未经授权访问客户数据，并防止一个租户的操作对另一个租户的服务造成负面影响。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-117">These controls are designed to safeguard against information leakage or unauthorized access to customer data across tenants and to prevent the actions of one tenant from adversely affecting the service for another tenant.</span></span>

<span data-ttu-id="e6c9f-118">客户内容在逻辑上隔离Microsoft 365 Azure AD Azure Active Directory (租户) 。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-118">Customer content is logically isolated within Microsoft 365 tenants using Azure Active Directory (Azure AD).</span></span> <span data-ttu-id="e6c9f-119">Microsoft 365身份验证不仅验证用户标识，还验证用户帐户属于的租户标识，从而阻止用户访问其租户环境外部的数据。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-119">User authentication in Microsoft 365 verifies not only the user identity, but also the tenant identity the user account is part of, preventing users from accessing data outside their tenant environment.</span></span> <span data-ttu-id="e6c9f-120">为了补充 Azure AD 的逻辑隔离，客户内容始终在其余和传输过程中进行加密。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-120">To supplement the logical isolation of Azure AD, customer content is always encrypted at rest and in transit.</span></span> <span data-ttu-id="e6c9f-121">个别服务也可能提供附加的租户隔离层，例如SharePoint、加密的数据库中的租户数据进行联机隔离。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-121">Individual services may also provide additional layers of tenant isolation, such as SharePoint Online isolation of tenant data in separate, encrypted databases.</span></span>

## <a name="how-does-microsoft-365-engineer-resilient-services-that-avoid-single-points-of-failure"></a><span data-ttu-id="e6c9f-122">如何Microsoft 365能够避免单点故障的复原服务？</span><span class="sxs-lookup"><span data-stu-id="e6c9f-122">How does Microsoft 365 engineer resilient services that avoid single points of failure?</span></span>

<span data-ttu-id="e6c9f-123">Microsoft 设计和构建云服务，以最大程度地提高可靠性，并最大程度地降低客户在正常运行时面临的故障和挑战带来的负面影响。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-123">Microsoft designs and builds cloud services to maximize reliability and minimize the negative effects on customers in the face of faults and challenges to normal operations.</span></span> <span data-ttu-id="e6c9f-124">此策略从连接地理位置分散的数据中心的网络设计开始。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-124">This strategy begins with the design of the network connecting our geographically distributed datacenters.</span></span> <span data-ttu-id="e6c9f-125">Microsoft 的网络体系结构包括直接互连和多个网络路径。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-125">Microsoft's network architecture includes direct interconnections and multiple network paths.</span></span> <span data-ttu-id="e6c9f-126">Microsoft 365服务利用此冗余自动路由故障周围的流量以提高服务质量。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-126">Microsoft 365 services leverage this redundancy to automatically route traffic around failures to improve service quality.</span></span>

<span data-ttu-id="e6c9f-127">在服务级别，Microsoft 365策略优先处理软件复原。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-127">At the service level, Microsoft 365's resilience strategy prioritizes software resiliency.</span></span> <span data-ttu-id="e6c9f-128">只要有可能，我们的服务将部署在主动/主动配置中，并自动进行服务运行状况监视，从而允许服务在无需人为干预的情况下从许多常见错误和故障中检测和恢复。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-128">Wherever possible, our services are deployed in active/active configurations with automated service health monitoring, allowing the service to detect and recover from many common faults and failures without human intervention.</span></span> <span data-ttu-id="e6c9f-129">除了主动/主动配置之外，Microsoft 365服务还通过确保服务部署在单独的容错区域中来增加容错能力，以防止一个区域中的错误影响其他区域的可用性。</span><span class="sxs-lookup"><span data-stu-id="e6c9f-129">In addition to active/active configurations, Microsoft 365 services increase fault-tolerance by ensuring the service is deployed in separate fault zones, preventing a fault in one zone from affecting the availability of other zones.</span></span>
