## <a name="how-does-microsoft-365-ensure-isolation-between-customer-tenants"></a><span data-ttu-id="f90d4-114">Microsoft 365 如何确保客户租户之间的隔离？</span><span class="sxs-lookup"><span data-stu-id="f90d4-114">How does Microsoft 365 ensure isolation between customer tenants?</span></span>

<span data-ttu-id="f90d4-115">Microsoft 的云服务基于以下假设：所有租户都有可能危害所有其他租户。</span><span class="sxs-lookup"><span data-stu-id="f90d4-115">Microsoft's cloud services are built on the assumption that all tenants are potentially hostile to all other tenants.</span></span> <span data-ttu-id="f90d4-116">为了正确地将租户彼此隔离，Microsoft 实施了各种隔离技术和控件。</span><span class="sxs-lookup"><span data-stu-id="f90d4-116">To properly isolate tenants from one another, Microsoft implements a variety of isolation technologies and controls.</span></span> <span data-ttu-id="f90d4-117">这些控件旨在防止信息泄露或跨租户未经授权访问客户数据，并防止一个租户的操作对另一个租户的服务造成负面影响。</span><span class="sxs-lookup"><span data-stu-id="f90d4-117">These controls are designed to safeguard against information leakage or unauthorized access to customer data across tenants and to prevent the actions of one tenant from adversely affecting the service for another tenant.</span></span>

<span data-ttu-id="f90d4-118">使用 Azure Active Directory 和 Azure AD (在 Microsoft 365 租户) 。</span><span class="sxs-lookup"><span data-stu-id="f90d4-118">Customer content is logically isolated within Microsoft 365 tenants using Azure Active Directory (Azure AD).</span></span> <span data-ttu-id="f90d4-119">Microsoft 365 中的用户身份验证不仅验证用户标识，还验证用户帐户包含的租户标识，从而阻止用户访问租户环境外部的数据。</span><span class="sxs-lookup"><span data-stu-id="f90d4-119">User authentication in Microsoft 365 verifies not only the user identity, but also the tenant identity the user account is part of, preventing users from accessing data outside their tenant environment.</span></span> <span data-ttu-id="f90d4-120">为了补充 Azure AD 的逻辑隔离，客户内容始终在其余和传输过程中进行加密。</span><span class="sxs-lookup"><span data-stu-id="f90d4-120">To supplement the logical isolation of Azure AD, customer content is always encrypted at rest and in transit.</span></span> <span data-ttu-id="f90d4-121">个别服务也可能提供附加的租户隔离层，例如 SharePoint Online 隔离独立、加密的数据库中的租户数据。</span><span class="sxs-lookup"><span data-stu-id="f90d4-121">Individual services may also provide additional layers of tenant isolation, such as SharePoint Online isolation of tenant data in separate, encrypted databases.</span></span>

## <a name="how-does-microsoft-365-engineer-resilient-services-that-avoid-single-points-of-failure"></a><span data-ttu-id="f90d4-122">Microsoft 365 如何设计避免单点故障的复原服务？</span><span class="sxs-lookup"><span data-stu-id="f90d4-122">How does Microsoft 365 engineer resilient services that avoid single points of failure?</span></span>

<span data-ttu-id="f90d4-123">Microsoft 设计和构建云服务，以最大程度地提高可靠性，并最大程度地降低客户在正常运行时面临的故障和挑战带来的负面影响。</span><span class="sxs-lookup"><span data-stu-id="f90d4-123">Microsoft designs and builds cloud services to maximize reliability and minimize the negative effects on customers in the face of faults and challenges to normal operations.</span></span> <span data-ttu-id="f90d4-124">此策略从连接地理位置分散的数据中心的网络设计开始。</span><span class="sxs-lookup"><span data-stu-id="f90d4-124">This strategy begins with the design of the network connecting our geographically distributed datacenters.</span></span> <span data-ttu-id="f90d4-125">Microsoft 的网络体系结构包括直接互连和多个网络路径。</span><span class="sxs-lookup"><span data-stu-id="f90d4-125">Microsoft's network architecture includes direct interconnections and multiple network paths.</span></span> <span data-ttu-id="f90d4-126">Microsoft 365 服务利用此冗余自动路由故障周围的流量，以提高服务质量。</span><span class="sxs-lookup"><span data-stu-id="f90d4-126">Microsoft 365 services leverage this redundancy to automatically route traffic around failures to improve service quality.</span></span>

<span data-ttu-id="f90d4-127">在服务级别，Microsoft 365 的恢复策略优先处理软件恢复。</span><span class="sxs-lookup"><span data-stu-id="f90d4-127">At the service level, Microsoft 365's resilience strategy prioritizes software resiliency.</span></span> <span data-ttu-id="f90d4-128">只要有可能，我们的服务将部署在主动/主动配置中，并自动进行服务运行状况监视，从而允许服务在无需人为干预的情况下从许多常见错误和故障中检测和恢复。</span><span class="sxs-lookup"><span data-stu-id="f90d4-128">Wherever possible, our services are deployed in active/active configurations with automated service health monitoring, allowing the service to detect and recover from many common faults and failures without human intervention.</span></span> <span data-ttu-id="f90d4-129">除了主动/主动配置之外，Microsoft 365 服务还通过确保服务部署在单独的容错区域中来增加容错能力，以防止一个区域中的错误影响其他区域的可用性。</span><span class="sxs-lookup"><span data-stu-id="f90d4-129">In addition to active/active configurations, Microsoft 365 services increase fault-tolerance by ensuring the service is deployed in separate fault zones, preventing a fault in one zone from affecting the availability of other zones.</span></span>

<span data-ttu-id="f90d4-130">数据复原通过保护 Microsoft 365 服务中数据的完整性和可用性来补充服务恢复能力。</span><span class="sxs-lookup"><span data-stu-id="f90d4-130">Data resiliency complements service resiliency by protecting the integrity and availability of data in Microsoft 365 services.</span></span> <span data-ttu-id="f90d4-131">我们的服务使用本地存储冗余和地理位置冗余将客户数据的副本复制到不同的容错区域。</span><span class="sxs-lookup"><span data-stu-id="f90d4-131">Our services use local storage redundancy and geo-redundancy to replicate copies of customer data into different fault zones.</span></span> <span data-ttu-id="f90d4-132">如果一个故障区域中的数据已损坏或丢失，可以在另一个故障区域中访问该数据，而不会丢失可用性。</span><span class="sxs-lookup"><span data-stu-id="f90d4-132">If data is corrupted or lost in one fault zone, it can be accessed in another fault zone without loss of availability.</span></span> <span data-ttu-id="f90d4-133">自动完整性检查自动还原受多种物理或逻辑损坏影响的数据。</span><span class="sxs-lookup"><span data-stu-id="f90d4-133">Automated integrity checking automatically restores data impacted by many kinds of physical or logical corruption.</span></span> <span data-ttu-id="f90d4-134">Microsoft 365 还为客户提供了还原 Exchange Online 和 SharePoint Online 中客户意外删除或修改的数据的工具。</span><span class="sxs-lookup"><span data-stu-id="f90d4-134">Microsoft 365 also provides customers with tools to restore data accidentally deleted or modified by the customer in Exchange Online and SharePoint Online.</span></span>

<span data-ttu-id="f90d4-140">Microsoft 365 体系结构定期并自动使用基于云的工具进行验证，以验证是否符合我们的安全原则，并持续测试隔离和恢复能力功能。</span><span class="sxs-lookup"><span data-stu-id="f90d4-140">Microsoft 365 architecture is validated regularly and automatically using cloud-based tools to verify alignment with our security principles and to continuously test isolation and resiliency features.</span></span> <span data-ttu-id="f90d4-141">体系结构验证可自动识别服务当前状态偏离所需状态的实例，并标记任何偏差以便进行审阅和缓解。</span><span class="sxs-lookup"><span data-stu-id="f90d4-141">Architectural validation works to automatically identify instances where the current state of the service has drifted from the desired state, flagging any deviations for review and mitigation.</span></span> <span data-ttu-id="f90d4-142">体系结构验证的目标是确保服务基础结构的管理功能继续正常工作。</span><span class="sxs-lookup"><span data-stu-id="f90d4-142">The goal of architecture validation is to ensure the security capabilities of our service infrastructure continue to function as expected.</span></span>
