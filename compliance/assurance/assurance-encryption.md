## <a name="what-role-does-encryption-play-in-protecting-customer-content"></a><span data-ttu-id="c6fbf-104">加密在保护客户内容方面扮演什么角色？</span><span class="sxs-lookup"><span data-stu-id="c6fbf-104">What role does encryption play in protecting customer content?</span></span>

<span data-ttu-id="c6fbf-105">大多数 Microsoft 商业云服务都是多租户的，这意味着客户内容可能存储在与其他客户相同的物理硬件上。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-105">Most Microsoft business cloud services are multitenant, meaning that customer content may be stored on the same physical hardware as that of other customers.</span></span> <span data-ttu-id="c6fbf-106">为了保护客户内容的机密性，Microsoft 365 使用一些最强大和最安全的加密协议对静态和传输中的所有数据进行加密。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-106">To protect the confidentiality of customer content, Microsoft 365 encrypts all data at rest and in transit with some of the strongest and most secure encryption protocols available.</span></span>

<span data-ttu-id="c6fbf-107">加密不能替代强访问控制。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-107">Encryption is not a substitute for strong access controls.</span></span> <span data-ttu-id="c6fbf-108">Microsoft 365 的访问控制策略 Zero Standing Access (ZSA) 保护客户内容免受 Microsoft 员工未经授权的访问。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-108">Microsoft 365's access control policy of Zero Standing Access (ZSA) protects customer content from unauthorized access by Microsoft employees.</span></span> <span data-ttu-id="c6fbf-109">加密通过保护存储位置的客户内容机密性和防止在 Microsoft 365 系统之间传输时或在 Microsoft 365 与客户之间传输时读取内容来补充访问控制。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-109">Encryption complements access control by protecting the confidentiality of customer content wherever it is stored and by preventing content from being read while in transit between Microsoft 365 systems or between Microsoft 365 and the customer.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-at-rest"></a><span data-ttu-id="c6fbf-110">Microsoft 365 如何加密其余数据？</span><span class="sxs-lookup"><span data-stu-id="c6fbf-110">How does Microsoft 365 encrypt data-at-rest?</span></span>

<span data-ttu-id="c6fbf-111">Microsoft 365 中所有客户内容都受一种或多种加密形式的保护。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-111">All customer content in Microsoft 365 is protected by one or more forms of encryption.</span></span> <span data-ttu-id="c6fbf-112">Microsoft 服务器使用 BitLocker 在卷级别加密包含客户内容的磁盘驱动器。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-112">Microsoft servers use BitLocker to encrypt the disk drives containing customer content at the volume-level.</span></span> <span data-ttu-id="c6fbf-113">BitLocker 提供的加密可保护客户内容，以防其他过程或控件 (例如，访问控制或硬件) 的回收可能会导致对包含客户内容的磁盘进行未经授权的物理访问。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-113">The encryption provided by BitLocker protects customer content in case of lapses in other processes or controls (for example, access control or recycling of hardware) that could lead to unauthorized physical access to disks containing customer content.</span></span>

<span data-ttu-id="c6fbf-114">Exchange Online、Microsoft Teams、SharePoint Online 和 OneDrive for Business 还在应用程序层使用服务加密来加密客户内容。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-114">Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business also use Service Encryption at the application layer to encrypt customer content.</span></span> <span data-ttu-id="c6fbf-115">服务加密提供权限保护以及强加密保护的管理功能。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-115">Service Encryption provides rights protection and management features on top of strong encryption protection.</span></span> <span data-ttu-id="c6fbf-116">它还允许在 Windows 操作系统和这些操作系统存储或处理的客户数据之间分离。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-116">It also allows for separation between Windows operating systems and the customer data stored or processed by those operating systems.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-in-transit"></a><span data-ttu-id="c6fbf-117">Microsoft 365 如何加密传输数据？</span><span class="sxs-lookup"><span data-stu-id="c6fbf-117">How does Microsoft 365 encrypt data-in-transit?</span></span>

<span data-ttu-id="c6fbf-118">Microsoft 365 产品和服务使用强传输协议（如 TLS）来防止未经授权的方在客户数据通过网络移动时窃听客户数据。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="c6fbf-119">传输数据的示例包括正在传递的邮件、联机会议中的对话或在数据中心之间复制的文件。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

<span data-ttu-id="c6fbf-120">在 Microsoft 365 中，每当用户设备与 Microsoft 服务器通信或 Microsoft 服务器与另一台服务器通信时，数据都被视为"正在传输"。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-120">In Microsoft 365, data is considered 'in transit' whenever a user's device is communicating with a Microsoft server or a Microsoft server is communicating with another server.</span></span> <span data-ttu-id="c6fbf-121">Exchange Online、SharePoint Online、Microsoft Teams 和 Office Online 均使用 TLS 来确保数据在传输过程中保持机密性。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-121">Exchange Online, SharePoint Online, Microsoft Teams, and Office Online all use TLS to ensure data remains confidential while in transit.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="c6fbf-122">Microsoft 365 如何管理用于加密的密钥？</span><span class="sxs-lookup"><span data-stu-id="c6fbf-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="c6fbf-123">强加密仅与用于加密数据的密钥一样安全。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-123">Strong encryption is only as secure as the keys used to encrypt data.</span></span> <span data-ttu-id="c6fbf-124">Microsoft 使用自己的安全证书加密传输数据的 TLS 连接。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-124">Microsoft uses its own security certificates to encrypt TLS connections for data-in-transit.</span></span> <span data-ttu-id="c6fbf-125">对于静态数据，受 BitLocker 保护的卷使用全卷加密密钥进行加密，该密钥使用卷主密钥进行加密，而卷主密钥又绑定到服务器中的受信任的平台模块 (TPM) 。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-125">For data-at-rest, BitLocker-protected volumes are encrypted with a full volume encryption key, which is encrypted with a volume master key, which in turn is bound to the Trusted Platform Module (TPM) in the server.</span></span> <span data-ttu-id="c6fbf-126">BitLocker 使用符合 FIPS 的算法来确保从不会以清晰方式通过线路存储或发送加密密钥。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-126">BitLocker uses FIPS-compliant algorithms to ensure that encryption keys are never stored or sent over the wire in the clear.</span></span>

<span data-ttu-id="c6fbf-130">具有控制自己的根加密密钥的要求的客户可以使用服务加密和客户密钥。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="c6fbf-131">使用客户密钥，客户可以使用本地硬件服务模块 (HSM) 或 AKV (生成自己的) 。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="c6fbf-132">客户根密钥存储在 AKV 中，可在 AKV 中用作加密客户邮箱数据或文件的密钥链之一的根。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="c6fbf-133">客户根密钥仅能由 Microsoft 365 服务代码间接访问，用于数据加密，并且不能由 Microsoft 员工直接访问。</span><span class="sxs-lookup"><span data-stu-id="c6fbf-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
