## <a name="response-management-process"></a><span data-ttu-id="58adf-139">响应管理过程</span><span class="sxs-lookup"><span data-stu-id="58adf-139">Response management process</span></span>

<span data-ttu-id="58adf-140">Microsoft 联机服务安全团队和服务团队协同工作，并采用相同方法处理安全事件，这基于 NIST 800-61 响应管理阶段：</span><span class="sxs-lookup"><span data-stu-id="58adf-140">Microsoft online services security teams and service teams work together on and take the same approach to security incidents, which is based on the NIST 800-61 response management phases:</span></span>

- <span data-ttu-id="58adf-141">**准备**：是指能够响应的组织准备工作，包括工具、流程、能力和准备情况。</span><span class="sxs-lookup"><span data-stu-id="58adf-141">**Preparation**: Refers to the organizational preparation that is needed to be able to respond, including tools, processes, competencies, and readiness.</span></span>
- <span data-ttu-id="58adf-142">**检测&** 分析：是指检测生产环境中的安全事件并分析所有事件以确认安全事件的真实性的活动。</span><span class="sxs-lookup"><span data-stu-id="58adf-142">**Detection & analysis**: Refers to the activity to detect a security incident in a production environment and to analyze all events to confirm the authenticity of the security incident.</span></span>
- <span data-ttu-id="58adf-143">**包含、抑制、** 恢复：是指根据上一阶段中完成的分析，为包含安全事件而采取的适当必需操作。</span><span class="sxs-lookup"><span data-stu-id="58adf-143">**Containment, eradication, recovery**: Refers to the required and appropriate actions taken to contain the security incident based on the analysis done in the previous phase.</span></span> <span data-ttu-id="58adf-144">在此阶段，可能还需要进行更多分析，以从安全事件完全恢复。</span><span class="sxs-lookup"><span data-stu-id="58adf-144">More analysis may also be necessary in this phase to fully recovery from the security incident.</span></span>
