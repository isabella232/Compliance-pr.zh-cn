# <a name="identity-and-access-management-overview"></a><span data-ttu-id="d27f6-103">标识和访问管理概述</span><span class="sxs-lookup"><span data-stu-id="d27f6-103">Identity and access management overview</span></span>

## <a name="how-does-microsoft-365-protect-production-systems-from-unauthorized-or-malicious-access"></a><span data-ttu-id="d27f6-104">Microsoft 365 如何保护生产系统免受未经授权的或恶意访问？</span><span class="sxs-lookup"><span data-stu-id="d27f6-104">How does Microsoft 365 protect production systems from unauthorized or malicious access?</span></span>

<span data-ttu-id="d27f6-105">Microsoft 365 旨在允许 Microsoft 工程师在不访问客户内容的情况下操作服务。</span><span class="sxs-lookup"><span data-stu-id="d27f6-105">Microsoft 365 is designed to allow Microsoft's engineers to operate the service without accessing customer content.</span></span> <span data-ttu-id="d27f6-106">默认情况下，Microsoft 365 工程师具有零长期访问 (ZSA) 访问客户内容，并且没有对生产环境的特权访问。</span><span class="sxs-lookup"><span data-stu-id="d27f6-106">By default, Microsoft 365 engineers have Zero Standing Access (ZSA) to customer content and no privileged access to the production environment.</span></span> <span data-ttu-id="d27f6-107">Microsoft 365 使用实时 (JIT) 、Just-Enough-Access (JEA) 模型，在支持 Microsoft 365 时需要提供对生产环境的临时特权访问。</span><span class="sxs-lookup"><span data-stu-id="d27f6-107">Microsoft 365 uses a Just-In-Time (JIT), Just-Enough-Access (JEA) model to provide service team engineers with temporary privileged access to production environments when such access is required to support Microsoft 365.</span></span> <span data-ttu-id="d27f6-108">JIT 访问模型将传统的持久管理访问替换为工程师根据需要请求临时提升为特权角色的过程。</span><span class="sxs-lookup"><span data-stu-id="d27f6-108">The JIT access model replaces traditional, persistent administrative access with a process for engineers to request temporary elevation into privileged roles when required.</span></span>

<span data-ttu-id="d27f6-109">工程师被分配到服务团队以支持生产服务通过标识管理工具请求服务团队帐户的资格 (IDM) 。</span><span class="sxs-lookup"><span data-stu-id="d27f6-109">Engineers assigned to a service team to support production services request eligibility for a service team account through the Identity Management Tool (IDM).</span></span> <span data-ttu-id="d27f6-110">资格申请触发了一系列人员检查，以确保工程师已通过所有云筛选要求、完成必要的培训，并获得了帐户创建前的适当管理批准。</span><span class="sxs-lookup"><span data-stu-id="d27f6-110">The request for eligibility triggers a series of personnel checks to ensure the engineer has passed all cloud screening requirements, completed necessary training, and received appropriate management approval prior to account creation.</span></span> <span data-ttu-id="d27f6-111">只有在满足所有资格要求后，才能为请求的环境创建服务团队帐户。</span><span class="sxs-lookup"><span data-stu-id="d27f6-111">Only after meeting all eligibility requirements can a service team account be created for the requested environment.</span></span>

<span data-ttu-id="d27f6-112">服务团队帐户不会授予任何长期管理员权限或客户内容的访问权限。</span><span class="sxs-lookup"><span data-stu-id="d27f6-112">Service team accounts do not grant any standing administrator privileges or access to customer content.</span></span> <span data-ttu-id="d27f6-113">当工程师需要其他访问权限来支持其 Microsoft 365 服务时，他们使用名为"密码箱"的访问管理工具请求临时提升的资源访问权限。</span><span class="sxs-lookup"><span data-stu-id="d27f6-113">When an engineer requires additional access to support their Microsoft 365 service, they request temporary elevated access to the resources they require using an access management tool called Lockbox.</span></span> <span data-ttu-id="d27f6-114">密码箱将提升的访问权限限制为完成分配的任务所需的最小特权、资源和时间。</span><span class="sxs-lookup"><span data-stu-id="d27f6-114">Lockbox restricts elevated access to the minimum privileges, resources, and time needed to complete the assigned task.</span></span> <span data-ttu-id="d27f6-115">如果授权审阅者批准 JIT 访问请求，则只会向工程师授予一个临时帐户，该帐户仅具有完成其分配的工作所需的权限。</span><span class="sxs-lookup"><span data-stu-id="d27f6-115">If an authorized reviewer approves the JIT access request, the engineer is granted a temporary account with only the privileges necessary to complete their assigned work.</span></span> <span data-ttu-id="d27f6-116">此临时帐户需要多重身份验证，在批准期限到期后自动删除。</span><span class="sxs-lookup"><span data-stu-id="d27f6-116">This temporary account requires multi-factor authentication and is automatically deleted after the approved period expires.</span></span>

## <a name="how-does-microsoft-365-handle-remote-access-to-production-systems"></a><span data-ttu-id="d27f6-117">Microsoft 365 如何处理对生产系统的远程访问？</span><span class="sxs-lookup"><span data-stu-id="d27f6-117">How does Microsoft 365 handle remote access to production systems?</span></span>

<span data-ttu-id="d27f6-118">Microsoft 365 系统组件位于地理位置上与运营团队分开的数据中心。</span><span class="sxs-lookup"><span data-stu-id="d27f6-118">Microsoft 365 system components are housed in datacenters geographically separated from the operations teams.</span></span> <span data-ttu-id="d27f6-119">数据中心人员对 Microsoft 365 系统没有逻辑访问权限。</span><span class="sxs-lookup"><span data-stu-id="d27f6-119">Datacenter personnel do not have logical access to Microsoft 365 systems.</span></span> <span data-ttu-id="d27f6-120">因此，Microsoft 365 服务团队人员通过远程访问管理环境。</span><span class="sxs-lookup"><span data-stu-id="d27f6-120">As a result, Microsoft 365 service team personnel manage the environment through remote access.</span></span> <span data-ttu-id="d27f6-121">需要远程访问以支持 Microsoft 365 的服务团队人员只有在获得授权经理批准后，才被授予远程访问。</span><span class="sxs-lookup"><span data-stu-id="d27f6-121">Service team personnel who require remote access to support Microsoft 365 are only granted remote access after approval from an authorized manager.</span></span> <span data-ttu-id="d27f6-122">所有远程访问都使用 FIPS 140-2 兼容 TLS 进行安全远程连接。</span><span class="sxs-lookup"><span data-stu-id="d27f6-122">All remote access uses FIPS 140-2 compatible TLS for secure remote connections.</span></span>

<span data-ttu-id="d27f6-123">Microsoft 365 使用安全管理工作站 (SAW) 服务团队远程访问，以帮助保护 Microsoft 365 环境免遭入侵。</span><span class="sxs-lookup"><span data-stu-id="d27f6-123">Microsoft 365 uses Secure Admin Workstations (SAWs) for service team remote access to help protect Microsoft 365 environments from compromise.</span></span> <span data-ttu-id="d27f6-124">这些工作站专门用于防止有意或无意丢失生产数据，包括锁定 USB 端口，以及将 SAW 上可用的软件限制到支持环境所需的操作。</span><span class="sxs-lookup"><span data-stu-id="d27f6-124">These workstations are specifically designed to prevent intentional or unintentional loss of production data, including locking down USB ports and limiting the software available on the SAW to what is required for supporting the environment.</span></span> <span data-ttu-id="d27f6-125">SAW 受到密切跟踪和监视，以检测和防止 Microsoft 工程师恶意或无意泄露客户数据。</span><span class="sxs-lookup"><span data-stu-id="d27f6-125">SAWs are closely tracked and monitored to detect and prevent malicious or inadvertent compromise of customer data by Microsoft engineers.</span></span>

## <a name="how-does-customer-lockbox-add-additional-protection-for-customer-content"></a><span data-ttu-id="d27f6-126">客户密码箱如何为客户内容添加其他保护？</span><span class="sxs-lookup"><span data-stu-id="d27f6-126">How does Customer Lockbox add additional protection for customer content?</span></span>

<span data-ttu-id="d27f6-127">客户可以通过启用客户密码箱来向其内容添加其他级别的访问控制。</span><span class="sxs-lookup"><span data-stu-id="d27f6-127">Customers can add an additional level of access control to their content by enabling Customer Lockbox.</span></span> <span data-ttu-id="d27f6-128">当密码箱提升请求涉及访问客户内容时，客户密码箱需要客户批准，这是审批工作流的最后一步。</span><span class="sxs-lookup"><span data-stu-id="d27f6-128">When a Lockbox elevation request involves access to customer content, Customer Lockbox requires approval from the customer as a final step in the approval workflow.</span></span> <span data-ttu-id="d27f6-129">这使组织可以选择批准或拒绝这些请求，并为客户提供直接访问控制。</span><span class="sxs-lookup"><span data-stu-id="d27f6-129">This gives organizations the option to approve or deny these requests and provides direct access control to the customer.</span></span> <span data-ttu-id="d27f6-130">如果客户拒绝客户密码箱请求，则对所请求内容的访问将被拒绝。</span><span class="sxs-lookup"><span data-stu-id="d27f6-130">If the customer rejects a Customer Lockbox request, access to the requested content is denied.</span></span> <span data-ttu-id="d27f6-131">如果客户在特定时间内不拒绝或批准请求，那么请求将自动过期，Microsoft 不会获得客户内容的访问权限。</span><span class="sxs-lookup"><span data-stu-id="d27f6-131">If the customer does not reject or approve the request within a certain period, then the request will expire automatically without Microsoft obtaining access to customer content.</span></span> <span data-ttu-id="d27f6-132">如果客户批准请求，Microsoft 对客户内容的临时访问将在分配完成疑难解答操作的时间到期后自动记录、审核和撤消。</span><span class="sxs-lookup"><span data-stu-id="d27f6-132">If the customer approves the request, then Microsoft's temporary access to customer content will be logged, auditable, and revoked automatically after the time assigned to complete the troubleshooting operation expires.</span></span>
