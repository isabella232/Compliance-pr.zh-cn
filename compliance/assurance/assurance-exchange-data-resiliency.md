## <a name="single-page-restore"></a><span data-ttu-id="294d0-177">单个页面还原</span><span class="sxs-lookup"><span data-stu-id="294d0-177">Single Page Restore</span></span> 
<span data-ttu-id="294d0-178">单页还原（也称为 " *页面修补*"）是一种自动过程，在正常的副本中由正常副本替换损坏的数据库页面。</span><span class="sxs-lookup"><span data-stu-id="294d0-178">Single page restore, also known as *page patching*, is an automatic process where corrupt database pages are replaced by healthy copies from a healthy replica.</span></span> <span data-ttu-id="294d0-179">损坏页面的修复过程取决于数据库副本是活动的还是被动的。</span><span class="sxs-lookup"><span data-stu-id="294d0-179">The repair process for a corrupt page depends on whether the database copy is active or passive.</span></span> <span data-ttu-id="294d0-180">当活动数据库副本遇到损坏的页面时，可以从其副本之一复制页面，前提是它所复制的页面是最新的。</span><span class="sxs-lookup"><span data-stu-id="294d0-180">When an active database copy encounters a corrupted page, it can copy a page from one of its replicas, provided the page it copies is up to date.</span></span> <span data-ttu-id="294d0-181">此过程是通过将页面请求放入日志流中完成的，这是邮箱数据库复制的基础。</span><span class="sxs-lookup"><span data-stu-id="294d0-181">This process is accomplished by putting a request for the page into the log stream, which is the basis of mailbox database replication.</span></span> <span data-ttu-id="294d0-182">一旦副本遇到页面请求，它就会通过将页面副本发送到请求数据库副本来做出响应。</span><span class="sxs-lookup"><span data-stu-id="294d0-182">As soon as a replica encounters the page request, it responds by sending a copy of the page to the requesting database copy.</span></span> <span data-ttu-id="294d0-183">单页还原还提供了主动的异步通信机制，可从副本中请求页面，即使副本当前处于脱机状态也是如此。</span><span class="sxs-lookup"><span data-stu-id="294d0-183">Single page restore also provides an asynchronous communication mechanism for the active to request a page from replicas, even if the replicas are currently offline.</span></span>
