## <a name="how-does-microsoft-365-centralize-and-report-on-audit-logs"></a><span data-ttu-id="f4939-120">Microsoft 365 如何集中并报告审核日志？</span><span class="sxs-lookup"><span data-stu-id="f4939-120">How does Microsoft 365 centralize and report on audit logs?</span></span>

<span data-ttu-id="f4939-121">许多不同类型的日志数据都从 Microsoft 365 服务器上载到名为"部署"的内部大数据计算服务。</span><span class="sxs-lookup"><span data-stu-id="f4939-121">Many different types of log data are uploaded from Microsoft 365 servers to an internal, big data computing service called Cosmos.</span></span> <span data-ttu-id="f4939-122">每个服务团队将审核日志从各自的服务器上载到"子场"数据库中进行聚合和分析。</span><span class="sxs-lookup"><span data-stu-id="f4939-122">Each service team uploads audit logs from their respective servers into the Cosmos database for aggregation and analysis.</span></span> <span data-ttu-id="f4939-123">此数据传输通过经过 FIPS 140-2 验证的 TLS 连接在批准的端口和协议上发生，该连接使用名为 Office 数据加载程序 (ODL) 的专有自动化工具。</span><span class="sxs-lookup"><span data-stu-id="f4939-123">This data transfer occurs over a FIPS 140-2-validated TLS connection on approved ports and protocols using a proprietary automation tool called the Office Data Loader (ODL).</span></span>

## <a name="how-does-microsoft-365-protect-audit-logs"></a><span data-ttu-id="f4939-127">Microsoft 365 如何保护审核日志？</span><span class="sxs-lookup"><span data-stu-id="f4939-127">How does Microsoft 365 protect audit logs?</span></span>

<span data-ttu-id="f4939-128">Microsoft 365 中用于收集和处理审核记录的工具不允许对原始审核记录内容或时间顺序进行永久或不可恢复的更改。</span><span class="sxs-lookup"><span data-stu-id="f4939-128">The tools used in Microsoft 365 to collect and process audit records do not allow permanent or irreversible changes to the original audit record content or time ordering.</span></span> <span data-ttu-id="f4939-129">仅授权人员对存储在"波斯"中的 Microsoft 365 数据的访问权限。</span><span class="sxs-lookup"><span data-stu-id="f4939-129">Access to Microsoft 365 data stored in Cosmos is restricted to authorized personnel.</span></span> <span data-ttu-id="f4939-130">Microsoft 365 将审核功能管理限制为负责审核功能的有限服务团队成员子集。</span><span class="sxs-lookup"><span data-stu-id="f4939-130">Microsoft 365 restricts the management of audit functionality to the limited subset of service team members that are responsible for audit functionality.</span></span> <span data-ttu-id="f4939-131">这些团队成员无法修改或删除"科科斯"的数据，并且记录并审核对"科科斯"的日志记录机制的所有更改。</span><span class="sxs-lookup"><span data-stu-id="f4939-131">These team members do not have the ability to modify or delete data from Cosmos, and all changes to logging mechanisms for Cosmos are recorded and audited.</span></span> <span data-ttu-id="f4939-132">审核日志的保留时间足够长，以支持事件调查和满足法规要求。</span><span class="sxs-lookup"><span data-stu-id="f4939-132">Audit logs are retained long enough to support incident investigations and meet regulatory requirements.</span></span> <span data-ttu-id="f4939-133">在审核日志中保留数据的确切时段由服务团队确定;大多数审核日志数据将保留 90 天或更长时间。</span><span class="sxs-lookup"><span data-stu-id="f4939-133">The exact period of audit log data retention in Cosmos is determined by the service teams; most audit log data is retained for 90 days or longer.</span></span>

## <a name="how-does-microsoft-365-protect-end-user-identifiable-information-that-may-be-captured-in-audit-logs"></a><span data-ttu-id="f4939-134">Microsoft 365 如何保护可能在审核日志中捕获的最终用户可识别信息？</span><span class="sxs-lookup"><span data-stu-id="f4939-134">How does Microsoft 365 protect end-user identifiable information that may be captured in audit logs?</span></span>

<span data-ttu-id="f4939-135">在将数据上载到Insings之前，ODL 应用程序使用推移服务模糊化包含客户数据（如租户信息和最终用户可识别信息）的任何字段，并使用哈希值替换这些字段。</span><span class="sxs-lookup"><span data-stu-id="f4939-135">Prior to uploading data into Cosmos, the ODL application uses a scrubbing service to obfuscate any fields that contain customer data, such as tenant information and end-user identifiable information, and replace those fields with a hash value.</span></span> <span data-ttu-id="f4939-136">将重写匿名日志和哈希日志，然后将这些日志上载到"均能"中。</span><span class="sxs-lookup"><span data-stu-id="f4939-136">The anonymized and hashed logs are rewritten and then uploaded into Cosmos.</span></span>
