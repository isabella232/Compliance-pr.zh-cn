# <a name="manage-gdpr-data-subject-requests-with-the-dsr-case-tool-in-the-microsoft-365-compliance-center"></a>使用 Microsoft 365 合规中心中的 DSR 案例工具管理 GDPR 数据主体请求

《欧盟一般数据保护条例》(GDPR) 旨在保护和授予欧盟境内的个人隐私权。 GDPR 使欧盟的个人（称为数据主体）有权访问、检索、更正、清除和限制其个人数据的处理。 按照 GDPR 的规定，个人数据是指已确定身份或可确定身份的自然人的任何相关信息。 个人向其组织发出的对其个人数据执行操作的正式请求，称为数据主体请求 (DSR)。 有关在 Office 365 中响应数据的 DSR 的详细信息，请参阅《[Office 365 数据主体请求指南](https://go.microsoft.com/fwlink/?linkid=871169 )》。
  
若要管理调查以响应组织中人员提交的 DSR，可以使用安全与合规中心中的 DSR 案例工具查找存储在以下位置的内容：
  
- 组织中的任何用户邮箱。 这包括 Microsoft Teams 中的 Skype for Business 对话和一对一聊天

- 与 Microsoft 365 组关联的所有邮箱以及 Microsoft Teams 中的所有团队邮箱

- 组织中的所有 SharePoint Online 网站和 OneDrive for Business 帐户

- 组织中的所有 Microsoft Teams 网站和 Microsoft 365 组网站

- Exchange Online 中的所有公用文件夹

你可以使用 DSR 案例工具：
  
- 为每个 DSR 调查创建单独的案例。

- 通过将人员添加为案例成员来控制谁有权访问 DSR 案例；只有成员可以访问案例，并且只能在合规中心的 **DSR 案例** 页面上查看其在案例列表中的案例。 此外，可为同一案例的不同成员分配不同的权限。 例如，可以只允许一些成员查看案例和搜索结果，而允许其他成员创建搜索和导出搜索结果。

- 使用内置搜索查询搜索与特定数据主体创建或上传的所有内容。

- 还可以修改内置搜索查询并重新运行搜索，以缩小搜索结果的范围。

- 添加与 DSR 案例关联的其他内容搜索。 这包括创建从 Office 漫游服务返回部分索引项和系统生成的日志的搜索。

- 导出数据以响应 DSR 访问或导出请求。

- 在 DSR 调查过程完成时删除案例集。这会删除所有搜索，并导出与案例集关联的作业。

以下是使用 DSR 案例工具管理 DSR 调查的高级流程：

[步骤 2：创建 DSR 案例并添加成员](#step-2-create-a-dsr-case-and-add-members)

[步骤 3：运行搜索查询](#step-3-run-the-search-query)

[步骤 4：导出数据](#step-4-export-the-data)

[（可选）步骤 5：修改内置搜索查询](#optional-step-5-revise-the-built-in-search-query)

[有关使用 DSR 案例工具的详细信息](#more-information-about-using-the-dsr-case-tool)
  
> [!IMPORTANT]
> 我们的工具允许管理员利用 DSR 案例工具中找到的内置搜索和导出功能，帮助他们执行 DSR 访问或导出请求。 该工具可助你尽最大努力地导出与数据主体提交的 DSR 请求相关的数据。 但是，请务必注意，搜索结果可能因数据主体或管理员操作而异，这些操作可能会影响项目是否被视为可供导出的“个人数据”。 例如，如果数据主体是最后一个修改并非他们本人创建的文件的人员，则搜索结果中可能不会返回该文件。 同样，管理员可以导出数据，而不包含部分索引项或所有版本的 SharePoint 文档。 因此，提供的工具有助于访问和导出数据请求；但是，结果受具体的管理员和数据主体使用方案的约束。
  
## <a name="step-1-assign-ediscovery-permissions-to-potential-case-members"></a>步骤 1：向潜在案例成员分配电子数据展示权限

默认情况下，全局管理员可以访问 Microsoft 365 合规中心中的 DSR 案例工具。 根据设计，其他用户（如数据隐私官、人力资源经理或参与 DSR 调查的其他人员）无权访问 DSR 案例工具，并且必须分配适当的权限才能访问该工具。 最简单的方法是前往合规中心的“**权限**”页面，并将用户添加到“电子数据展示管理器”角色组。 你还必须分配这些权限，以便可以将其添加为在步骤 2 中创建的 DSR 案例的成员。 
  
有关分步说明，请参阅 [分配电子数据展示权限](/microsoft-365/compliance/assign-ediscovery-permissions)。
  
> [!NOTE]
> 默认情况下，全局管理员（或合规中心的组织管理角色组的其他成员，没有导出内容搜索结果必需的权限（请参阅本文中的步骤 4）。 若要解决此问题，管理员可以将自己添加为电子数据展示管理器角色组的成员。 
  
## <a name="step-2-create-a-dsr-case-and-add-members"></a>步骤 2：创建 DSR 案例并添加成员

下一步是创建 DSR 案例。 创建案例时，可以选择启动内置搜索，也可以在不启动搜索的情况下创建案例。 以下过程介绍了在不启动搜索的情况下创建案例，然后向你演示了如何向案例添加成员。
  
1. 转到 <https://compliance.microsoft.com>，使用工作或学校帐户登录。

2. 在 Microsoft 365 合规中心的左窗格中，选择 **数据主体请求**。

3. 在 **数据主体请求** 页面上，单击 **创建案例**。

4. 在 **新建 DSR 案例** 向导页面上，为案例指定名称，键入可选说明，然后单击 **"下一步"**。 案例名称在你的组织中必须保持唯一。

    > [!TIP]
    > 请考虑在新案例的名称和/或说明中添加提交你正在调查的 DSR 请求的人员的姓名。 请注意，只有本案例的成员（和电子数据展示管理员）才能在 **数据主体请求** 页面上的案例列表中看到此案例。
  
5. 在“**请求详细信息**”页面上，“**数据主题（提交此请求的人员）**”下方，选择要查找和导出其数据的人员，然后点击“**下一步**”。

6. 在“**确认案例设置**”页面上，可以更改案例名称和说明，并选择其他数据主体。 如果不需要更改，请点击“**下一步**”。

    将会显示一个页面，确认已创建新的 DSR 案例。

    ![开始搜索或关闭“新建 DSR 案例”页面](../media/b5e62c2c-cafe-4a8d-a38c-789ed9f9ccbd.png)
  
    此时，你可以执行以下两项操作中的一项：

    a. 点击“**显示搜索结果**”，启动搜索。 此为默认选项。 步骤 3 中会介绍选择此选项时行的内置搜索，以及返回的结果。

    b. 点击“**完成**”，在不启动内置搜索的情况下关闭新的 DSR 案例。 选择此选项时，新的 DSR 案例将显示在“**数据主体请求**”页面上。

7. 点击“**完成**”，以便可以转到新的 DSR 案例并向其添加成员。

8. 在“**数据主体请求** 页面上，点击创建的 DSR 案例的名称。

9. 在”**管理此案例**“浮出页面上，“**管理成员**”下方，点击“**添加**”。

    在“**用户**”下方，将显示分配了相应电子数据展示权限的人员列表。 在步骤 1 中分配了电子数据展示权限的人员，将会显示在此列表中。

10. 选择要添加为 DSR 案例成员的人员，点击“**添加**”，然后保存更改。

    还可以将角色组添加为 DSR 案例成员，方法是点击“**管理角色组**”下的”**添加**”。 

## <a name="step-3-run-the-search-query"></a>步骤 3：运行搜索查询

创建 DSR 案例并添加成员后，运行与案例关联的内置搜索。此默认搜索查询会执行以下操作：
  
- 在组织中的所有邮箱中，搜索数据主体发送或接收的所有电子邮件项目。 这是通过使用  *参与者*  电子邮件属性来实现的，该属性会搜索电子邮件消息中，所有人员字段中的数据主体。 该属性会返回 **发件人**、**收件人**、**抄送** 和 **密件抄送** 字段中包含数据主体的项目。 Exchange Online 中的公用文件夹也会搜索数据主体发送或接收的邮件。

- 在组织中的所有站点中，搜索数据主体创建或上传的文档和项目。 这是通过使用下列站点属性实现的：

  - “*作者*”属性返回数据主体在 Office 文档的作者字段中列出的项。 即使文档由其他人复制和上传，该值仍会保留。

  - “*创建者*”属性返回数据主体创建或上传的项。

下面是创建 DSR 案例时自动创建的内置搜索的关键字查询。
  
```powershell
participants:"<email address>" OR author:"<display name>" OR createdby:"<display name>"
```

例如，如果数据主体的姓名是 Ina Leonte，则关键字查询如下所示：
  
```powershell
participants:"ina@contoso.com" OR author:"Ina Leonte" OR createdby:"Ina Leonte"
```

 **要运行 DSR 案例的内置搜索，请执行以下操作：**
  
1. 在 Microsoft 365 合规中心，单击 **数据主体请** 求，在步骤 2 中选择创建的案例，然后单击 **"打开案例"**。

2. 选择页面顶部的“**搜索**”选项卡，然后点击创建 DSR 案例时创建的内置搜索旁边的复选框。 搜索的名称与 DSR 案例相同。 

3. 在搜索浮出控件页中，单击 **重新运行** 以开始搜索。

4. 搜索完成后，点击“**预览结果**”预览搜索结果。 有关详细信息，请参阅[预览搜索结果](/microsoft-365/compliance/content-search#preview-search-results)。

    > [!TIP]
    > 还可以查看搜索查询统计信息，以查看搜索返回的邮箱和网站项数量，以及包含与搜索查询匹配的项目的主要内容位置。 有关详细信息，请参阅[查看与搜索相关的信息和统计信息](/microsoft-365/compliance/content-search#view-information-and-statistics-about-a-search)。 
  
可以编辑内置搜索查询，更改搜索的内容位置，然后重新运行搜索。 请参阅[步骤 5](#optional-step-5-revise-the-built-in-search-query) 以了解更多信息。
  
## <a name="step-4-export-the-data"></a>步骤 4：导出数据

运行内置搜索后，可以导出搜索结果。 或者，在导出数据之前，修改查询以减少搜索结果数量。 有关缩小搜索结果范围的详细信息，请参阅步骤 5。
  
导出搜索结果时，可将邮箱项目下载为 PST 文件或单独的邮件。 当你从 SharePoint 和 OneDrive 帐户导出内容时，将导出本地 Office 文档副本。 搜索结果中包含一个包含与每个导出项有关信息的结果文件。 有关导出的更多详细信息，请参阅[导出内容搜索结果](/microsoft-365/compliance/export-search-results)。
  
> [!NOTE]
> 默认情况下，全局管理员（或合规中心中组织管理角色组的其他成员）没有导出内容搜索结果必需的权限。 若要解决此问题，管理员可以将自己添加为电子数据展示管理器角色组的成员。 
  
用于导出数据的计算机须满足下列系统要求：
  
- 32 位或 64 位版本的 Windows 7 和更高版本
    
- Microsoft .NET Framework 4.7
    
- 支持的浏览器：
    
  - Microsoft Edge
    
    或
    
  - Microsoft Internet Explorer 10 和更高版本
    
    > [!NOTE]
    > Microsoft 不开发面向 ClickOnce 应用程序的第三方扩展或加载项。 不支持使用具有第三方扩展或加载项的浏览器导出数据。 
  
 **若要从 DSR 案例中的内置搜索导出数据，请执行以下操作：**
  
1. 在 Microsoft 365 合规中心，单击 **数据主体请** 求，在步骤 2 中选择创建的案例，然后单击 **"打开案例"**。

2. 选择页面顶部的“**搜索**”选项卡，然后点击创建 DSR 案例时创建的内置搜索旁边的复选框。

3. 在浮出页面上，单击“**导出结果**”。 

4. 在“**导出结果**”页面上，为 DSR 导出请求选择以下建议选项。 

    ![配置导出设置](../media/25416b79-57da-46a1-ae07-e640602a8fa4.png)
  
    a. 在“**输出选项**”下方，选择第一个选项（**所有项（不包括具有无法识别格式的项）已加密，或由于其他原因未编制索引**）以仅导出已编制索引的项目。 不建议从内置搜索中导出部分索引项的原因是，这样做会连同其他用户的已编入索引的部分项目一起导出。 若要仅导出数据主体的部分索引项，建议创建单独的搜索。 有关详细信息，请参阅“有关使用 DSR 案例工具的更多信息”部分的“[导出部分索引项](#exporting-partially-indexed-items)”。

    b. 在“**将 Exchange 内容导出为** 下方，选择第三个选项，**包含单个文件夹中所有邮件的 One PST 文件**。 由于某些结果可能属于源自其他用户邮箱的项目，因此，此选项只列出单个文件夹中的项目，而不指示实际邮箱，并且是取消复制下一项中建议的结果时使用的最佳选项。 此选项还允许数据主体按时间顺序查看项目（按发送日期排序项目），而无需导航到每个项目的原始邮箱文件夹结构。

    c. 选择 **启用重复数据删除** 选项，以排除重复的电子邮件。 我们建议使用此选项，因为内置搜索会搜索组织中的所有邮箱。 因此，如果在已搜索的邮箱中找到同一封邮件的多个副本，则此选项意味着将只导出邮件的一个副本。 此选项会一起导出单个文件夹中一个 PST 文件中的邮件，为 DSR 导出请求提供最佳用户体验。 Results.csv 导出报表会列出发现重复邮件的所有位置。

    也可以选择“**包含 SharePoint 文档的版本**”选项，以导出所有版本的 SharePoint 和 OneDrive 文档。 这要求为文档库启用版本控制。 此选项有助于确保导出所有相关数据。

5. 选择导出设置后，点击“**导出**”。

    搜索结果已准备下载是指它们被上传至 Microsoft 云中，你的组织的 Azure 存储区域。 后续步骤将会介绍如何将此数据下载到本地计算机。

6. 点击“**导出**”选项卡以显示创建的导出作业。 导出作业的名称与搜索名称最后带有“**_Export**”的相应搜索相同。

7. 点击刚刚创建的导出作业，以显示导出浮出页面。 此页显示有关搜索的信息，例如要导出的项的大小和总数，以及已传输到 Azure 存储区域的项的百分比。 点击“**刷新**”，更新上传状态信息。 

8. 在“**导出密钥**”下，点击“**复制到剪贴板**”。 在步骤 11 中，将使用此密钥下载搜索结果。

9. 点击导出浮出页面顶部的![导出搜索结果图标](../media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) **下载结果**。 

10. 在页面底部的弹出窗口中，点击“**打开**”以打开“**电子数据展示导出工具**”。 如果是首次下载搜索结果，则会安装 **电子数据展示导出工具**。 

11. 在“**电子数据展示导出工具**”中，将在步骤 8 中复制的导出密钥粘贴在相应的框中。

12. 单击“**浏览**”指定要下载搜索结果文件的位置。 

    > [!NOTE]
    > 由于磁盘活动量很大（读取和写入），应将搜索结果下载到本地磁盘驱动器；勿将其载到映射的网络驱动器或其他网络位置。 
  
13. 单击“启动”将搜索结果下载到计算机。 

    **电子数据展示工具** 显示有关导出过程的状态信息，包括要下载的剩余项的估计数量（和大小）。 导出过程完成后，你可以在文件下载的位置访问它们。 有关下载内容搜索结果时包含的报告的详细信息，请参阅“导出内容搜索结果”中的“[更多信息](/microsoft-365/compliance/export-search-results#more-information)”。 

导出数据后，搜索结果和导出报告位于与 DSR 案例同名的文件夹中。 包含邮箱项的 PST 文件位于名为“**Exchange**”的子文件夹中。 网站中的文档和其他项目位于名为“**SharePoint**”的子文件夹中。
  
## <a name="optional-step-5-revise-the-built-in-search-query"></a>（可选）步骤 5：修改内置搜索查询

运行内置搜索后，可以对其进行修改以缩小范围，返回更少的搜索结果。 你可以添加向查询添加条件来实现。 条件将通过 **AND** 运算符在逻辑上连接至关键字查询。 这意味着，项必须同时满足关键字查询和添加的所有条件，才能在搜索结果中返回。 这就是条件如何帮助缩小结果范围的原理。 如果将两个或多个唯一性条件（指定不同属性的条件）添加到搜索查询中，这些条件将在逻辑上使用 **AND** 运算符来连接。 这意味着仅返回满足所有条件（除了关键字查询以外）的项目。 如果向单个条件中添加多个值（用逗号或分号分隔），这些值将通过 **OR** 运算符来连接。 这意味着如果这些项包含条件中指定的任何属性值，则返回这些项。
  
以下是可添加到 DSR 案例内置搜索查询中的一些示例条件。 搜索查询中使用的实际属性的名称显示为括号。
  
- **文件类型 (`filetype`)** – 文档或文件的扩展名。 使用此条件搜索由特定 Office 应用程序（如 Word、Excel 和 OneNote）创建的文档和文件。 

- **邮件类型 (`kind`)** – 指定要搜索的电子邮件项的类型。 例如，可以使用语法  `kind:email OR kind:im` 仅返回电子邮件和 Skype for Business 对话，或 Microsoft Teams 中的一对一聊天。

- **合规性标记 (`compliancetag`)** – 指定分配给电子邮件或文档的标签。 此条件会返回使用特定标签分类的项。 标签用于对电子邮件和文档进行分类，以实现数据治理，并根据标签定义的分类加强实施保留规则。 此条件对于 DSR 调查非常有用，因为组织可能使用了标签对数据隐私相关内容或者包含个人数据或敏感信息的内容进行了分类。 对于此条件的值，请使用完整的标签名称或带有通配符的标签名称的第一部分。 有关详细信息，请参阅[了解保留策略和保留标签](/microsoft-365/compliance/retention)。

有关可以在 DSR 案例工具中使用的条件的列表和描述，请参阅“内容搜索的关键词查询和搜索条件”文章中的“[搜索条件](/microsoft-365/compliance/keyword-queries-and-search-conditions#search-conditions)”。 
  
### <a name="changing-the-content-locations-that-are-searched"></a>更改搜索的内容位置

除了修改 DSR 案例的内置搜索之外，还可以更改搜索的内容位置。 如前所述，内置搜索会搜索组织中的每个邮箱和网站，以及任何 Exchange Online 公用文件夹。 例如，可以将搜索范围缩小为仅搜索数据主体的邮箱、OneDrive 帐户和所选 SharePoint 网站。 如果选择搜索特定网站，则必须添加要搜索的每个网站。
  
要修改要搜索的内容位置：请执行以下操作：
  
1. 打开要更改内容位置的内置搜索。

2. 在搜索查询的“**位置**”下方，点击“**特定位置**”选项旁边的“**修改**”。 

    ![点击“修改”，更改内置搜索查询的内容位置](../media/d66f7ba7-b71f-4ff5-a030-460ff02e3123.png)
  
    将显示“**修改位置**”浮出页面。 页面中会显示内置搜索中内容位置的说明，以及有关修改所搜索位置的一些信息。 

    ![修改位置浮出页面。](../media/56c033f6-6735-46ba-abb2-a263a2b79836.png)
  
    a. 浮出页面顶部，邮箱部分“**全选**”下方的切换键处于选中状态，这表示搜索所有邮箱。 若要缩小搜索范围，请点击切换键取消选中，然后点击 **选择用户、组或团队**”，并选择要搜索的具体邮箱。

    b. 浮出页面中部，网站部分“**全选**”下方的切换键处于选中状态，这表示搜索所有网站。 若要将搜索范围缩小到所选网站，请取消选择切换键，然后点击“**选择网站**”。 必须添加要搜索的每个具体网站，包括数据主体的 OneDrive 帐户。

    c. “Exchange 公用文件夹”部分中的切换键处于选中状态，这意味着搜索所有 Exchange 公用文件夹。 你只能要么仅搜索所有 Exchange 公用文件夹，要么完全不搜索。 无法选择具体的 Exchange 公用文件夹进行搜索。

3. 如果修改内置搜索中的内容位置，请点击“**保存&amp;运行**“以重新启动搜索。 

> [!NOTE]
> 搜索所有邮箱位置或仅特定邮箱，导出搜索结果时，保存至用户邮箱的其他 Office 365 应用程序数据将包括在内。 此数据将不会包括在估计的搜索结果中，并且也不可用于预览。 但在导入和下载搜索结果时，此数据将包括在内。 有关在用户邮箱中存储数据的应用程序的详细信息，请参阅[存储在 Exchange Online 邮箱中的内容](/microsoft-365/compliance/what-is-stored-in-exo-mailbox)。

### <a name="exporting-data-from-the-office-roaming-service"></a>从 Office 漫游服务导出数据

可以使用 DSR 案例工具搜索并导出由 Office 漫游服务生成的使用情况数据。 漫游是一种可以存储 Office 相关设置，如 Office 主题、自定义词典、语言设置、开发者模式和自动更正的服务。

来自 Office 漫游服务的数据存储在数据主体邮箱中的隐藏文件夹中，该文件夹位于 Exchange Online 邮箱的非人际邮件（非 IPM）子树中。 这意味着当用户使用 Outlook 或其他邮件客户端访问其邮箱时，用户的视图中不会显示此类数据。 有关隐藏文件夹的详细信息，请参阅 [MAPI 隐藏文件夹](https://go.microsoft.com/fwlink/?linkid=872758)。
  
可以创建单独的内容搜索（并将其与 DSR 事例相关联），以返回数据主体邮箱中的 Office 漫游服务使用情况数据。 此数据不包含在搜索统计信息中，并且不可预览。 但可以将其导出，然后提供给数据主体以响应 DSR 导出请求。
  
从 Office 漫游服务导出数据时，数据将保存到位于 **ApplicationDataRoot** 文件夹中的单独文件夹中，该文件夹位于以数据主体电子邮件地址命名的文件夹下。 此数据导出为附加到电子邮件的 JSON 文件（与 XML 或 TXT 文件类似，可供人工读取的文本文件）。 目前，此文件夹采用全局唯一标识符 (GUID) 命名：**1caee58f-eb14-4a6b-9339-1fe2ddf6692b**。 在 DSR 案例工具的未来版本中，GUID 将替换为实际应用程序的名称。 

**若要搜索并导出 Office 漫游服务数据，请执行以下操作：**
  
1. 在 Microsoft 365 合规中心中，单击 **数据主体请求**，然后单击要为其导出使用情况数据的数据主体的 DSR 案例旁边的"**打开案例**"。 

2. 单击页面顶部的“**搜索**”选项卡，然后点击![添加图标](../media/ITPro-EAC-AddIcon.gif) **引导式搜索**。

3. 在 **命名搜索 **页面上，点击“** 取消**”。 

4. 在“**搜索查询**”下，“**类型**”条件中，选中“**Office 漫游服务**”旁边的复选框。 

    ![选中 Office 漫游服务复选框以导出使用情况数据](../media/O365_DSRCase_SDSDataExport1.png)
  
    “**类型**”条件（即电子邮件类）应是搜索查询中的唯一项。 可以删除 **关键字** 框或留空。 

5. 在“**位置**”下，确保选中“**特定位置**”，然后点击“**修改**”。

6. 在“**修改位置**”浮出页面（邮箱部分）顶部，点击“**选择用户、组或团队**”。

7. 在“**编辑位置**”页面上，点击“**选择用户、组或团队**”，选择数据主体的邮箱，然后保存所选内容。 

8. 点击“**保存并运行**”，然后命名搜索并保存。

    搜索即启动。

 **若要导出 Office 漫游服务数据，请执行以下操作：**
  
1. 当上个步骤中创建的搜索完成后，点击页面顶部的“**搜索**”选项卡，然后点击搜索旁边的复选框。 可能必须点击“![刷新](../media/165fb3ad-38a8-4dd9-9e76-296aefd96334.png) **刷新**”才能显示搜索。

2. 在浮出页面上，单击“**导出结果**”。

3. 在“**导出结果**”页面上，选择建议的选项以导出使用情况数据。 

    ![导出 Office 漫游服务使用情况数据时的导出选项](../media/470a7d1e-eeae-4b42-95aa-15cb82ce2f68.png)
  
    a. 在“**输出选项**”下方，选择第一个选项（**所有项（不包括具有无法识别格式的项）已加密，或由于其他原因未编制索引**）以仅导出已编制索引的项目。

    b. 在“**将 Exchange 内容导出为** 下方，选择第二个选项，**一个 PST 文件包含所有邮件**。

    c. 剩余的导出选项保持未选中状态。

4. 选择导出设置后，点击“**导出**”。

    搜索结果已准备下载是指它们被上传至 Microsoft 云中的 WindowsAzure 存储区域。 后续步骤将会介绍如何将此数据下载到本地计算机。

5. 点击“**导出**”选项卡以显示创建的导出作业。 导出作业的名称与搜索名称最后带有“**_Export**”的相应搜索相同。 

6. 点击刚刚创建的导出作业，以显示导出浮出页面。 

7. 在“**导出密钥**”下，点击“**复制到剪贴板**”。 在步骤 10 中，将使用此密钥下载搜索结果。

8. 点击导出浮出页面顶部的![导出搜索结果图标](../media/47205c65-babd-4b3a-bd7b-98dfd92883ba.png) **下载结果**。 

9. 在页面底部的弹出窗口中，点击“**打开**”以打开“**电子数据展示导出工具**”。 如果是首次下载搜索结果，则会安装 **电子数据展示导出工具**。 

10. 在“**电子数据展示导出工具**”中，将在步骤 7 中复制的导出密钥粘贴在相应的框中。

11. 单击“**浏览**”指定要下载搜索结果文件的位置。 

    > [!NOTE]
    > 由于磁盘活动量很大（读取和写入），应将搜索结果下载到本地磁盘驱动器；勿将其载到映射的网络驱动器或其他网络位置。 
  
12. 单击“启动”将搜索结果下载到计算机。 

    **电子数据展示工具** 显示有关导出过程的状态信息，包括要下载的剩余项的估计数量（和大小）。 导出过程完成后，可以在 Outlook 中打开 Exchange PST 文件，然后转到 **ApplicationDataRoot** 文件夹以访问漫游服务的子文件夹。 

    如前所述，包含使用情况数据的 JSON 文件会附加到邮件中。 若要查看 JSON 文件，点击一封邮件，然后打开附件中的 JSON 文件。
  
### <a name="exporting-partially-indexed-items"></a>导出部分索引项

建议不要从创建 DSR 案例时创建的内置搜索中导出部分索引项（也称为未编制索引的项目）。 这是因为搜索结果很可能包含组织中其他用户的部分索引项，而不仅仅是数据主体的部分索引项。 我们建议创建一个单独的内容搜索，该搜索与 DSR 案例相关联，旨在仅导出与数据主体相关的部分索引项。
  
下面是导出部分索引项的高级流程。导出后，可以查看它们，以确定项目是否响应 DSR 访问或导出请求。
  
1. 打开 DSR 案例，并在“**搜索**”页面上创建搜索。

2. 使用以下条件配置搜索查询，以及要搜索的内容位置：

    - 使用空的关键字查询。这样做会返回搜索的内容位置中的所有项。

    - 仅搜索数据主体的 Exchange Online 邮箱及其 OneDrive 帐户。

3. 运行搜索并完成后，可以导出并下载搜索结果（如[步骤 4](#step-4-export-the-data) 中所述）。 使用以下设置导出部分索引项。

    - 在“**输出选项**”下方，选择第三个选项（**仅具有无法识别格式、已加密，或由于其他原因未编制索引的项**）以仅导出部分索引项。

    - 在“**将 Exchange 内容导出为**”下，你可以根据偏好选择任何选项。

    - 选择“**包含 SharePoint 文档的版本**”选项导出文档的版本（如果某个版本已部分编制索引）。

有关详细信息，请参阅：
  
- [内容搜索中的部分索引项](/microsoft-365/compliance/partially-indexed-items-in-content-search)

- [导出部分索引项](/microsoft-365/compliance/export-search-results#exporting-partially-indexed-items)

### <a name="searching-and-exporting-data-from-microsoft-teams-and-microsoft-365-groups"></a>搜索并从 Microsoft Teams 和 Microsoft 365 组中导出数据

属于 Microsoft Teams 中聊天列表的对话（称为“团队聊天”或“一对一聊天”）将存储在参与该聊天的用户的 Exchange Online 邮箱中。 此外，用户在一对一聊天中共享的文件存储在该用户的 OneDrive 帐户中。 由于内置搜索会搜索组织中的所有邮箱和 OneDrive 帐户，因此在 DSR 案例中内置搜索会返回聊天会话中共享的团队聊天和文档（由数据主体创建或上传的）。
  
此外，Teams 频道中的对话（也称为“频道消息”）存储在与某个团队关联的邮箱中。 数据主体参与的这些类型的对话也由内置搜索返回，因为会搜索与 Teams 关联的所有邮箱。 此外，数据主体在 Teams 频道中共享的文件存储在团队的 SharePoint 网站上。 数据主体创建或上传的文件由 DSR 案例中的内置搜索返回，因为搜索中包含与 Teams 关联的网站。
  
同样，与 Microsoft 365 组对应的邮箱和 SharePoint 网站也包含在内置搜索中。 这意味着将返回数据主体发送或接收的电子邮件，以及数据主体创建或上传的文件。 
  
有关使用内容搜索在 Microsoft Teams 和 Microsoft 365 组中搜索项目，或查看如何获取成员列表的详细信息，请参阅“[Microsoft 365 中的内容搜索](/microsoft-365/compliance/content-search-reference#searching-microsoft-teams-and-microsoft-365-groups)”中的“搜索 Microsoft Teams 和 Microsoft 365 组”部分。
  
### <a name="searching-exchange-public-folders"></a>Exchange 公用文件夹

DSR 案例中的内置搜索将仅返回数据主体发送到启用邮件的公用文件夹，或其他人发送到公用文件夹并抄送给数据主体的电子邮件。 不会返回数据主体发布到公用文件夹的邮件。 若要搜索数据主体发布到公用文件夹的项目，可以单独创建一个内容搜索，用于搜索数据主体发布到公用文件夹的任何项目。
  
以下是搜索数据主体发布到公用文件夹的项目的高级流程。 
  
1. 打开 DSR 案例，并在“**搜索**”页面上创建搜索。 

2. 使用以下条件配置搜索查询，以及要搜索的内容位置：

   - 在“**关键字**”框中，使用以下搜索查询： 

     ```powershell
     itemclass:ipm.post AND "<email address of the data subject>"
     ```

   - 搜索所有 Exchange 公用文件夹

   - 运行搜索并完成后，可以导出并下载搜索结果（如[步骤 4](#step-4-export-the-data) 中所述）。 使用以下设置导出部分索引项。
